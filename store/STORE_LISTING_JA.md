## 詳しい説明（最大 4000 文字）

dIAry は、声で書けるシンプルな音声日記アプリです。
録音した音声をテキスト化して、日々の記録を続けやすくします。

【主な機能】
・音声録音（マイク）
・Google Gemini API による文字起こし/要約（ユーザーが入力した API キーを利用）
・日記データは端末内に保存（ローカル保存）
・Google Drive（AppDataFolder）へのバックアップ/復元（任意・Google ログイン）
・広告表示（AdMob）

【使い方】

1. 録音して日記を作成
2. 必要に応じて、設定から Gemini API キーを入力
3. バックアップが必要なら Google でログインして Drive に保存

【プライバシー】
・開発者のサーバーに個人情報を収集・保管しません。
・日記データは端末内に保存されます。
・文字起こし/要約を使う場合、音声データは Google（Gemini API）へ送信されます。
・Google Drive バックアップを使う場合、バックアップデータはユーザーの Google Drive（AppDataFolder）に保存されます。

プライバシーポリシー:
https://hidenobunagai.github.io/diary/PRIVACY_POLICY.html

【注意事項】
・文字起こし/要約機能はネットワーク接続が必要です。
・Gemini API の利用には Google 側の利用規約/料金が適用される場合があります。

## 画像アセット（Play Console）

### 1) アプリのアイコン（必須）

- 形式: PNG または JPEG
- サイズ: 512 × 512
- 容量: 1MB 以下

### 2) フィーチャー グラフィック（必須）

- 形式: PNG または JPEG
- サイズ: 1024 × 500

### 3) 携帯電話のスクリーンショット（必須）

- 推奨枚数: 4 枚以上（最低 2 枚 / 最大 8 枚）
- 形式: PNG または JPEG
- 1 枚あたり: 8MB 以下
- アスペクト比: 16:9 または 9:16
- 縦横のサイズ: 320〜3840 px

## おすすめのスクショ構成（そのまま撮れば OK）

1. ホーム（今日の入力/導線が分かる画面）
2. 録音中/録音完了（マイク利用が分かる画面）
3. 文字起こし結果/要約（Gemini 連携が伝わる画面）
4. 履歴一覧（継続利用のイメージ）
5. 設定（バックアップ/復元の導線が分かる画面）

※ 個人情報（メール・トークン・日記本文の生データなど）が写らないよう、テストデータで撮影推奨。

## スクショの取り方（Windows）

### A) 端末の通常スクショ（最短）

- Android: 電源 + 音量下

撮った画像は「Google フォト」同期や USB 転送で PC に取り込み。

### B) ADB で取得（画面が多い場合に便利）

1. Android SDK Platform Tools をインストール
2. 端末で「開発者向けオプション」→「USB デバッグ」を ON
3. PowerShell で実行:

```powershell
adb devices
adb shell screencap -p /sdcard/s1.png
adb pull /sdcard/s1.png .
```
